# 基於大語言模型的食物影像營養估算系統

本專案實作一款專為台灣飲食環境打造的「AI 營養估算系統」，結合生成式AI的視覺推理能力與官方營養資料庫的精確度，實現「拍照即時分析、精準紀錄」的目標。  
⋇此存放庫整理自團隊大三專題上學期部分成果，部分功能尚未完成，例如通知提醒功能。

---

## 專題動機

隨著現代人對健康管理意識提升，飲食紀錄成為關鍵需求。在設計系統前，我們分析了多款市面飲食紀錄與營養估算應用（如智抗糖、健談健管家），發現多數操作繁瑣、初次使用需花較長時間設定。  
本專題旨在開發結合 AI 視覺辨識與衛福部官方資料庫的智慧化系統，提供使用者精準且快速的營養分析工具。

---

## 功能特色

-  **AI 智慧影像估算**：  
採用 Google Gemini 2.5 Flash 模型進行影像辨識。先由 AI 進行食材辨識與語意指令解析（如「半飯」運算），再自動對接內建的「衛福部食品營養資料庫」進行查表校正，有效修正數值誤差，提供精準且即時的營養分析結果。

- **個人化健康目標設定**：  
依據一般包裝食品營養標示標準，設定通用的成人每日建議攝取量。當用戶輸入個人生理數值後，系統會依據BMR公式與活動係數，自動計算個人專屬的每日目標熱量與三大營養素建議量。

- **帳號與權限管理**：  
採用 Firebase Authentication 處理帳號相關操作，如登入、註冊、寄送驗證信等操作。後端採用 FastAPI 框架，透過後端驗證帳號權限，確保使用者身分不可由前端任意竄改。

- **多平台免登入、免安裝**：  
前端採用 Flutter 框架同步開發多平台應用，除手機 App 外，也支援網頁版。使用者無需安裝 App 或註冊帳號，即可快速使用核心功能，提升系統便利性與普及率。

## 下學期預計新增功能

- **通知提醒**：
加入通知提醒使用者如早餐未紀錄，或是顯示家人傳送提醒訊息。

- **週報月報**：
目前統計報表只提供單日的統計，預計加入周報、月報或自訂統計範圍，幫助使用者快速了解特定區間內飲食狀況。

- **家庭共享(群組)**：
目前初步測試後(程式碼尚未加入)已實現單向的資料共享，在設定中加入照顧者後，照顧者可以查看被照顧者飲食狀況。
