# 基於大語言模型的食物影像營養估算系統

本專案實作一款專為台灣飲食環境打造的「AI 營養估算系統」，結合生成式AI的視覺推理能力與官方營養資料庫的精確度，實現「拍照即時分析、精準紀錄」的目標。  
⋇此存放庫整理自團隊大三專題上學期部分成果，部分功能尚未完成，例如通知提醒功能。

---

## 專題動機

隨著現代人對健康管理意識提升，飲食紀錄成為關鍵需求，然而現有市面上的應用程式普遍存在「手動輸入繁瑣」與「缺乏在地化資料庫」等痛點。傳統的純 AI 模型多受限於西式飲食訓練資料，對台灣在地美食（如滷肉飯、便當）辨識準確度低且易產生數值幻覺。本專題旨在開發一套結合生成式 AI 視覺辨識與衛福部官方資料庫的智慧 化系統，解決跨文化飲食辨識誤差，提供使用者精準、快速且符合台灣法規標準的營養分析工具。

---

## 功能特色

-  **AI 智慧影像估算**：  
採用 Google Gemini 2.5 Flash 模型進行影像辨識。先由 AI 進行食材辨識與語意指令解析（如「半飯」運算），再自動對接內建的「衛福部食品營養資料庫」進行查表校正，有效修正數值誤差，提供精準且即時的營養分析結果。

- **個人化健康目標設定**：  
依據一般包裝食品營養標示標準，設定通用的成人每日建議攝取量。當用戶輸入個人生理數值後，系統會依據BMR公式與活動係數，自動計算個人專屬的每日目標熱量與三大營養素建議量。

- **帳號與權限管理**：  
採用 Firebase Authentication 處理帳號相關操作，如登入、註冊、寄送驗證信等操作。後端採用 FastAPI 框架，透過後端驗證帳號權限，確保使用者身分不可由前端任意竄改
